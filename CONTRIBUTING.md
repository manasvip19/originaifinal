# Contributing to OriginAI

Thank you for contributing.

Our goal is to accelerate innovation by connecting research and entrepreneurship.

---

# Development Philosophy

We value:

* Simplicity
* Clarity
* Reliability
* Explainability
* Innovation

---

# Repository Workflow

1. Fork repository
2. Create branch
3. Implement changes
4. Open pull request

---

# Branch Naming

feature/research-agent

feature/startup-generator

fix/pdf-parser

docs/readme-update

---

# Commit Standards

Examples:

feat: add startup scoring

fix: resolve upload validation bug

docs: improve user manual

refactor: optimize market agent

---

# Coding Standards

## Frontend

* Reusable components
* Consistent styling
* Responsive design

## Backend

* Modular architecture
* Typed interfaces
* Clean APIs

## AI

* Prompt versioning
* Validation checks
* Confidence scoring

---

# Documentation Standards

Every feature should document:

* Purpose
* Inputs
* Outputs
* Limitations

---

# Pull Request Checklist

* Code compiles
* Tests pass
* Documentation updated
* No secrets committed

---

# Reporting Issues

Include:

* Description
* Reproduction steps
* Expected behavior
* Actual behavior
* Screenshots

---

# Code Review Criteria

Reviewers evaluate:

* Correctness
* Maintainability
* Security
* Documentation quality
