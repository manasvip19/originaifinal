# Industry Mapping

Overview
- Map extracted discoveries to applicable industries and sectors.

Problem Statement
- Raw discoveries need contextualization to find business domains.

Goals
- Produce 3–5 industry mappings per discovery with short rationales and confidence.

User Stories
- As an entrepreneur, I want to see which industries could adopt a discovery.

Inputs
- Discovery items, keywords, taxonomy (NAICS-lite or custom tags).

Outputs
- Industry list per discovery: industry tag, rationale, estimated adoption painpoints.

Functional Requirements
- Maintain a small taxonomy mapping table in Postgres.
- Use semantic similarity between discovery and industry descriptions via embeddings.
- Provide human-readable rationale generated by OpenAI.

Non-Functional Requirements
- Mapping latency under 5s per discovery for MVP.

Workflow
- For each discovery: compute embedding -> query industry descriptions -> generate rationale with prompt -> return mapping.

AI Components
- OpenAI embeddings + chat; vector DB for taxonomy lookups.

Edge Cases
- Ambiguous discoveries mapping to many industries: rank by confidence and present top 3.

Success Metrics
- Relevance judged by team: >75% top-1 correct on sample set.

Acceptance Criteria
- Each discovery returns at least one plausible industry with rationale in the UI.
